## How Can I Contribute?

### üêõ Reporting Bugs

Before creating bug reports, please check existing issues to avoid duplicates. When creating a bug report, include:

- Clear, descriptive title
- Detailed steps to reproduce
- Expected vs actual behavior
- Screenshots if applicable
- Browser and OS information

### üí° Suggesting Features

Feature suggestions are welcome! Please:

- Use a clear, descriptive title
- Provide detailed description of the proposed feature
- Explain why this feature would be useful
- Include mockups or examples if possible

### üîß Pull Requests

1. Fork the repo and create your branch from `main`
2. Make your changes
3. Test thoroughly in multiple browsers
4. Update documentation if needed
5. Write a clear commit message
6. Submit the pull request!

## Code Style Guidelines

- Use consistent indentation (2 spaces)
- Keep functions small and focused
- Add comments for complex logic
- Use meaningful variable names
- Follow existing code patterns

## Feature Ideas

Here are some features we'd love to see:

- **Export functionality** - Download as PDF, TXT, or Markdown
- **User presence indicators** - Show who's currently typing
- **Themes** - Light/dark mode, different color schemes
- **Archive system** - Browse and search old pages
- **Rich text** - Basic formatting options
- **Private rooms** - Password-protected collaborative spaces
- **User cursors** - See where other people are typing in real-time
- **Undo/redo** - History management
- **Mobile improvements** - Better touch interactions
- **Accessibility** - Screen reader support, keyboard navigation

## Questions?

Feel free to open an issue with the tag `question` or reach out to the maintainers.
